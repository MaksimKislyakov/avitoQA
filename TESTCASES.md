# Создание объявления: POST /api/1/item

## Позитивные сценарии

Тест-кейс 1: Успешное создание объявления со всеми валидными полями
    Предусловия: нет
    Шаги: отправить POST запрос с валидным JSON: {"name": "Телефон", "price": 50000, "sellerId": 123322, "statistics": {"likes": 10, "viewCount": 100, "contacts": 3}}
    Ожидаемый результат: статус 200 OK. В ответе возвращается созданный объект с уникальным id (строка) и датой createdAt
    Статус: Passed

Тест-кейс 2: Создание объявления с нулевой ценой (бесплатно)
    Предусловия: нет
    Шаги: отправить POST запрос со всеми обязательными полями, где "price": 0
    Ожидаемый результат: статус 200 OK. Объявление создано, цена сохранена как 0
    Статус: Failed (см. BUG-1)

Тест-кейс 3: Создание объявления с нулевыми значениями статистики
    Предусловия: нет
    Шаги: отправить POST запрос, где внутри "statistics" значения "likes", "viewCount", "contacts" равны 0
    Ожидаемый результат: статус 200 OK. Объявление успешно создано.
    Статус: Failed (см. BUG-2)

## Негативные сценарии

Тест-кейс 4: Отправка запроса с отсутствующими обязательными полями
    Предусловия: нет
    Шаги: отправить POST запрос без поля sellerId (или name, price)
    Ожидаемый результат: статус 400 Bad Request. Сообщение об ошибке валидации, что поле обязательно
    Статус: passed

Тест-кейс 5: Отправка запроса с неверным типом данных
    Предусловия: нет
    Шаги: отправить price как строку ("15000") или sellerId как строку
    Ожидаемый результат: Статус 400 Bad Request. Ошибка типа данных.
    Статус: failed (см. BUG-3)

Тест-кейс 6: Создание объявления с отрицательной ценой
    Предусловия: нет
    Шаги: отправить POST запрос, где price: -1
    Ожидаемый результат: статус 400 Bad Request, "message": "цена не может быть отрицательной"
    Статус: failed (см. BUG-4)

Тест-кейс 7: Отправка невалидного JSON (Syntax Error)
    Предусловия: нет
    Шаги: отправить тело запроса с пропущенной запятой или кавычкой
    Ожидаемый результат: Статус 400 Bad Request
    Статус: passed
