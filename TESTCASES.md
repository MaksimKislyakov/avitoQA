# 1. Создание объявления: POST /api/1/item

## Позитивные сценарии

Тест-кейс 1: Успешное создание объявления со всеми валидными полями
    Предусловия: нет
    Шаги: отправить POST запрос с валидным JSON: {"name": "Телефон", "price": 50000, "sellerId": 123322, "statistics": {"likes": 10, "viewCount": 100, "contacts": 3}}
    Ожидаемый результат: статус 200 OK. В ответе возвращается созданный объект с уникальным id (строка) и датой createdAt
    Статус: failed (cм. BUG-1)

Тест-кейс 2: Создание объявления с нулевой ценой (бесплатно)
    Предусловия: нет
    Шаги: отправить POST запрос со всеми обязательными полями, где "price": 0
    Ожидаемый результат: статус 200 OK. Объявление создано, цена сохранена как 0
    Статус: failed (см. BUG-2)

Тест-кейс 3: Создание объявления с нулевыми значениями статистики
    Предусловия: нет
    Шаги: отправить POST запрос, где внутри "statistics" значения "likes", "viewCount", "contacts" равны 0
    Ожидаемый результат: статус 200 OK. Объявление успешно создано.
    Статус: failed (см. BUG-3)
