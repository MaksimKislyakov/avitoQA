BUG-1: Ошибка валидации поля price при значении 0
    Связанный тест-кейс: TK-2 (Создание объявления с нулевой ценой)
    Приоритет: high
    Эндпоинт: POST /api/1/item
    Описание: при передаче в JSON значения 0 для ключа price, сервер возвращает ошибку валидации, как если бы поле отсутствовало
    Ожидаемый результат: 200 OK, успешное создание объявления
    Фактический результат: 400 Bad Request, тело: {"message": "поле price обязательно"}

    Приоритет: high
    Эндпоинт: POST /api/1/item
