BUG-1: Ошибка валидации поля price при значении 0
    Связанный тест-кейс: TK-2 (Создание объявления с нулевой ценой)
    Приоритет: high
    Эндпоинт: POST /api/1/item
    Описание: при передаче в JSON значения 0 для ключа price, сервер возвращает ошибку валидации, как если бы поле отсутствовало
    Ожидаемый результат: 200 OK, успешное создание объявления
    Фактический результат: 400 Bad Request, тело: {"message": "поле price обязательно"}

BUG-10: Успешное создание объявления без заголовка Content-Type
    Связанный тест-кейс: ТК-27 (Отправка POST без заголовка Content-Type: application/json)
    Приоритет: medium
    Эндпоинт: POST api/1/item
    Описание: сервер успешно обрабатывает POST-запрос с телом в формате JSON, даже если заголовок Content-Type отсутствует или установлен в text/plain. Ожидается строгая валидация заголовков.
    Ожидаемый результат: cтатус 400 Bad Request или 415 Unsupported Media Type
    Фактический результат: cтатус 200 ОК, объявление создано
